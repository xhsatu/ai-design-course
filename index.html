你在跟什么说话

Claude、GPT、Gemini…
只是聪不聪明的差别？

AI 能力（模型）= 见过多少东西 × 怎么被训练

不是聪明程度的差别，是训练出来的本质不同。
我们说的「模型」，就是 AI 背后真正在运作的那个东西。

互动：有同学用过多个 AI 吗？能感受到区别吗？

三个顶尖模型，三种本质差异

Claude

有内核、有原则、会反驳。
擅长深度推理和复杂判断。

GPT

能干、全面、生态最广。
接入产品最多，最会「干活」。

Gemini

和 Google 全家桶打通。
多模态突出，支持超长文本。

跟不同模型说话，就像跟不同特质的人说话。你需要知道对方是谁。

实操：同一个问题问一下 Claude / 你手机里的 AI，分享差异

你怎么跟它说话

你每说一句话，
就是在发一个 Prompt

少量 Prompt → 模糊的输出
丰富 Prompt → 精准的输出

你补充的信息越多，输出就越精准。

实操：同一个问题，第一次不给背景，第二次给足背景

互动：有同学知道吗？

揭秘

客户端帮你做了
一件事

// 把所有对话打包，每次都重新发给 LLM
你发第 1 条
你发第 2 条 → 客户端把 1+2 打包重新发
你发第 3 条 → 客户端把 1+2+3 打包重新发

这个打包的内容，就是 Context（上下文）

是端上给它加了 Context，它才变成了「有记忆」。

但有上限

这叫 Context Window
（上下文窗口）

云端算力有限，必然存在上限。
聊久了会感觉记不住事——

不是真的忘，是内容太多，
模型的注意力已经覆盖不到了
互动：大家有遇到过「它忘了」的情况吗？

解决方案

有一个文件，
叫 CLAUDE.md

正常对话 Context 从零开始
TA 不认识你
有 CLAUDE.md 永远在 Context 里
TA 一直认识你

你写进去的东西，它每次对话都会读到。

实操：让 Claude Code 打开 CLAUDE.md 看看里面有什么

但是

TA 现在认识你了。

但光是「认识」
有什么用？

我们来看看——它能不能真正替你做一件事。

实操

把这段 Prompt 发给 TA

Prompt 基于你对我的了解，帮我生成一个属于我的 App。

这个 App 的定位和功能，必须完全基于你对我这个人的理解。不要问我做什么，你来决定。

最重要的一点：视觉风格必须体现我这个人的特质——配色、字体、间距、质感，每一个设计决策都应该能说明「这是为这个人做的」，而不是通用模板。风格和色彩是表达我是谁的主要语言。

技术：HTML+CSS+JS 单文件，设置 viewport，系统字体栈，字阶清晰。
设计：给设计师看的，标准要高，每个细节都要认真处理。
